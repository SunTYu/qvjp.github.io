从今天起，我们要为INLOW实现内存管理。第一步实现GDT，即全局描述表。

内存模型

我们知道，在实模式下，我们访问内存是直接访问线性地址，而在保护模式，由于地址空间变大，不能直接使用线性地址，x86提供了两种内存访问模式：分段（Segmented）、分页（Paged）。

分段

作为线性地址的替代，分段内存模型将内存分为一些或大或小的段。比如上一篇实现printf时，我们使用到了0xB8000---彩色视频内存地址，我们可以定义一个段，开始地址是0xB8000，比如它是08h段，那可以用段加偏移的方式，即08h:0002h来访问0xB8002。分段的限制是我们最多只能分8192个段，不过这已经足够了。我们可以给每个段设置不同的读写权限，并且段可以重叠。这篇博客要实现的就是上边所说。

分页

分页是另一种内存模型，常用语虚拟内存。这里不多说，下篇博客详细阐述。

Global Descriptor Table

Global Descriptor Table（GDT）是x86的一个用来告诉CPU内存分段情况的结构，下边详细介绍GDT。

结构

GDT Entry

每一个段对应GDT中一个项，它们的结构如下：

如图，GDT每个条目都是64位长，其中：

  • 0-15位和16-19位共20位用来表示最大地址单元，如果下边的Flags中的Gr设置为1，正好可以表示4GB的地址大小。
  • 16-39位和56-63位共32位用来表示段开始位置的线性地址。

  • 40-47和52-55位表示权限，有点复杂，具体结构如下图：

    如果所示，权限位共有8位，分别表示：

    • AC: Accessed bit，设置为0即可，CPU在访问后会设置此位为1
    • RW: Readable bit/Writable bit，
      代码段可读，数据段可写
      并且代码段永远不可写，数据段永远可读
    • DC: Direction bit/Conforming bit，
      对数据选择子是方向位，当DC = 0，段向上增长，DC = 1，段向下增长
      对代码选择子是位，当DC = 1，允许设定的或更低权限执行，DC = 0，只能设定的权限等级执行
    • Ex: Executable bit，1是代码，0是数据
    • DT: Descriptor Type，如果是数据段或代码段则设置为1
    • Privl: Privilege bit，2 bit 0 = highest(kernel)，3 = lowest(user applications)
    • Pr: Present bit，1 bit 所有有效的selectors这一位都设置为1

  • 52-55位表示Flags，结构如下：

    • SZ: Size bit，SZ = 0，16位保护模式，SZ = 1，32位保护模式
    • Gr: Granularity bit，Gr = 0，粒度为1 bytes，Gr = 1，粒度为4kB
GDT Selector

引用GDT中的段描述符所描述的段，是通过一个16位的Selector---段选择子来完成的，结构如下：

如上图，GDT Selector共16位，分为3部分：

  • 0-1两位用来表示特权级 00为最高特权级，11为最低特权级
  • 第二位用来标示此选择子为GDT或LDT，若为0则GDT，1为LDT
  • 2-15是GDT Entry的索引，这也解释了上边所说为什么最多只能有8192个段
GDT Descriptor

我们使用汇编指令LGDT来加载GDT，它需要我们提供GDT的位置及大小的描述，这个描述的结构如下：

如上图，GDTR的共48位，分为两部分，其中

  • 0-15位是Size，是GDT的大小，由于最多有8192个段，而GDT中每项是8 bytes，也就是GDT最大为65536个bytes
  • 16-47位是偏移量，也就是GDT的线性地址
