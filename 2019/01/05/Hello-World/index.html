Hello World From INLOW

发表于 2019-01-05 | 更新于 2019-02-17 | 分类于 INLOW ， Develop | 评论数： | 阅读次数：
本文字数： 8.5k | 阅读时长 ≈ 21 分钟

这篇博客记录INLOW的第一次实践，即简单的在屏幕上输出一串字母 — Hello World。

首先要说明的是，INLOW不是从头开发，是站在巨人的肩膀上，它没有自己实现一个编程语言、编译器，也没有自己写bootloader，这里用到的技术有：

  • GNU Compiler Collection（GCC）来编译高级语言到汇编。
  • GNU Assembler 来将汇编代码汇编成机器码。
  • GNU Linker 来将各个机器码链接成最终的内核。
  • C语言作为内核的开发语言。
  • GRUB作为bootloader，来加载32位内核。
  • ELF作为可执行格式，使可以控制内核的位置和加载方式。

实施之前

开始写代码之前，我们先理一下电脑开机到操作系统被加载的过程。

  • BIOS，计算机通电后，便是去主板上执行这段固定的程序，BIOS是基本输入输出系统的缩写。
    • 硬件自检，BIOS首先进行硬件自检，若没有问题，则进行下一步。
    • 启动顺序，这里BIOS需要知道下一步要执行的程序在哪，所以涉及到启动顺序，下图是我笔记本的启动顺序选项卡。
  • 主引导记录（MBR），BIOS按照启动顺序找到第一个存储设备，读取该设备第一个扇区，也就是最开始的512字节。如果这512个字节最后两个字节是0x55和0xAA，说明这个设备是可以用来启动的，若不是，则去找下一个设备。而最前边这512个字节，就叫做主引导记录（MBR）。
    • 主引导记录结构，主引导记录分为3部分，首先1-446字节是调用下一步的机器码，447-510字节是分区表，511-512是0x55和0xAA。
    • 分区表，分区表的长度是64个字节，每项16字节，所以只能有4项，这就是一个硬盘之能有4个主分区的原因。而每个项中又由6部分组成，这里不详细展开，但这里要知道的是这4个分区只能有一个是激活分区。
  • 硬盘启动，到这里根据上边的流程，把控制权交给了某个设备的某个分区，但具体启动又分为下边三种可能：
    • 卷引导记录，上边说过，四个分区只能有一个是激活的，计算机会读取这个激活的分区的第一个扇区，也就是卷引导记录（Volume boot record）。VBR会告诉计算机操作系统在分区里的位置，这样就可以加载操作系统了。
    • 扩展分区和逻辑分区，硬盘越来越大，四个主分区已经不够了，这就发明了扩展分区。规定4个分区中仅有1个可以被定义为扩展分区，扩展分区中又分为多个分区，这些分区被称为逻辑分区，逻辑分区中还可以包含逻辑分区，理论上是没有上限的。但很少有操作系统安装在扩展分区。
    • 启动管理器，这种情况下，计算机读取MBR前446个字节机器码后，不再把控制权交给某个分区，而是运行事先安装的bootloader，通常是GRUB。bootloader可以去找到操作系统，然后加载。
  • 操作系统，操作系统接管硬件控制权。

上边所说其实也只是一种情况，UEFI和GPT又是另一种思路了。但这里写这些的目的只是要明白，我们的内核是要被bootloader所识别并加载的，即BIOS->Bootloader->Kernel。所以如何让bootloader识别内核是我们要做的第一件事。

两种运行环境

在构建交叉编译器一章我们知道了C运行有两种环境，一种是Freestanding，另一种是Hosted，前者表示我们没有C标准库可用，任何东西都要自己提供。而后者就是我们经常编写的用户态程序，有标准库和各种有用的运行时特性。
然而，有些头文件不是C标准库的一部分，而是编译器所提供的。因此在Freestanding模式下也是可用的，比如下边我们用到的<stdint.h>和<stddef.h>分别提供了intx_t、uintx_t和size_t。
